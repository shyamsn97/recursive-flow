
Boids Simulation — Cross-file Contract (plain HTML + JS modules)

Files and required interfaces:

1) output/boids-simulation/index.html
   - A minimal HTML page that:
     - Includes: <canvas id="boids"></canvas>
     - Loads module script: <script type="module" src="js/main.js"></script>
     - Links stylesheet: <link rel="stylesheet" href="css/style.css">
     - Sets page title: "Boids Simulation"
     - Provides a container for the fullscreen canvas (no external libs).

2) output/boids-simulation/css/style.css
   - Styles to make canvas full-window:
     - html, body: margin:0; padding:0; width:100%; height:100%; background:#0b1020; overflow:hidden;
     - canvas: display:block; width:100vw; height:100vh; image-rendering:pixelated;

3) output/boids-simulation/js/utils.js
   - Must export the following functions (exact names/signatures):
     export function randRange(min, max)            // float in [min, max)
     export function clamp(x, lo, hi)               // clamp number
     export function wrap(value, max)               // wrap to [0, max) toroid
     export function hsl(h, s, l)                   // return "hsl(h s% l%)" or "hsl(h, s%, l%)"
     export function len(vx, vy)                    // vector length
     export function limit(vx, vy, max)             // limit vector magnitude, returns [vx, vy]

   - All functions should be pure and not rely on DOM.

4) output/boids-simulation/js/boid.js
   - Must export the Boid class with exact signatures:
     export class Boid {
       constructor(x, y, vx, vy, hue)
       update(boids, w, h, params)   // compute acceleration and integrate velocity/position; toroidal wrap
       draw(ctx)                     // draw a triangle or circle with color derived from hue
     }
   - Behavior:
     - Maintain this.x, this.y, this.vx, this.vy, this.hue
     - Use Reynolds rules: separation, alignment, cohesion
     - Respect params: { perceptionRadius, separationRadius, maxSpeed, maxForce, alignWeight, cohWeight, sepWeight }
     - Apply wrapping across w/h using utils.wrap
     - Keep boids "fast" by maxSpeed (suggest 180–260 px/s). Hue can vary per boid.

5) output/boids-simulation/js/sim.js
   - Must export:
     export const DEFAULT_PARAMS = {
       perceptionRadius: number,
       separationRadius: number,
       maxSpeed: number,
       maxForce: number,
       alignWeight: number,
       cohWeight: number,
       sepWeight: number
     }
     export class Simulation {
       constructor(canvas)           // expects an HTMLCanvasElement
       resize()
       addBoids(n)
       step(dt)                      // dt in seconds
       draw()
       run()                         // requestAnimationFrame loop
     }
   - Responsibilities:
     - Hold: this.canvas, this.ctx, this.boids (array), this.params, this.lastTime
     - On resize, match canvas size to devicePixelRatio for crisp rendering
     - step(dt) updates all boids with current width/height/params
     - draw() clears and draws all boids
     - run() animates using rAF with time delta in seconds

   - Imports:
     import { Boid } from './boid.js'

6) output/boids-simulation/js/main.js
   - Must:
     import { Simulation } from './sim.js'
     - Query #boids canvas, create a Simulation(canvas)
     - Add hundreds of boids (>= 300), e.g., sim.addBoids(400)
     - Start the loop via sim.run()
     - Handle window resize: update canvas size via sim.resize()

General constraints:
- Plain JS modules (ESM). No external libraries.
- Keep code readable and efficient; naive O(n^2) neighbor search is acceptable.
- Use requestAnimationFrame for smooth animation.
- Keep colors vivid and boids move quickly.
- Ensure files are written exactly at the specified paths.
